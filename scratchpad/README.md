# Scratchpad

Lowest-friction entry point: anything rough that fits in git.

## Conventions

- `{yyyy}/{yyyy}-{mm}-{dd}.md` — the day's captures, appended.
- `{yyyy}/{yyyy}-{mm}-{dd}/` — standalone artifacts born that day; create only
  when needed.
- `{yyyy}/{yyyy}-{mm}-{dd}-{slug}/` — a named thread (multi-file, multi-day);
  the date is the start date, never renamed. Larger threads graduate to their
  own repo.
- Full-date basenames: a filename should identify itself outside its directory.
- Records, not state: no TODOs or checkboxes here. Edit freely while a thread
  is warm; once cold, supersede with a newer dated entry rather than rewrite.
- Polished pieces graduate to `notebook/`.
